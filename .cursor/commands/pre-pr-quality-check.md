# Pre-PR Quality Check

Prepare this branch for a pull request.

1. Summarize intent and key changes vs the base branch.
2. Confirm: plan/docs for large changes, tests for behavior changes, no secrets, local-first / no unapproved cloud.
3. Run the relevant lint/type/test commands that actually apply; report only observed results.
4. List remaining risks and a short test plan for reviewers. Do not fabricate CI or test outcomes.
