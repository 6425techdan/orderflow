# Architecture Review

Review the current change or plan for architectural fit with OrderFlow (6425 Reliability Lab).

1. Inspect relevant existing modules, boundaries, and docs before judging.
2. Check: local-first constraint, smallest credible design, coupling/blast radius, failure modes.
3. Flag over-engineering, missing ownership boundaries, and docs that do not match the design.
4. Output: findings (severity-ordered), assumptions, and concrete recommendations. Do not implement unless asked.
